# AIOS Strong Team Blueprint

## Decision

Keep the five visible workflow agents, but make each one an accountable multidisciplinary squad. Forge coordinates them and explains the evidence; the owner retains consequential approval.

## Seven ranks

1. Apprentice — learns within a bounded task.
2. Junior — assists with verified mission work.
3. Specialist — owns a defined engineering discipline.
4. Senior — coordinates complex specialist handoffs.
5. Lead — assures squad outcomes and risk.
6. Principal — influences architecture across the guild.
7. Guild Master — sets strategy under owner governance.

Rank is based on completed mission experience, not elapsed time or unverified activity.

## Combined squads

| Squad | Primary responsibility | Embedded disciplines |
| --- | --- | --- |
| Router | Product requirements and systems architecture | Product discovery, UX requirements, OSINT authorization |
| Planner | Solution architecture and source intelligence | Architecture, source mapping, threat modeling |
| Builder | Full-stack, API and cloud engineering | Frontend, backend/API, Cloudflare integration |
| Tester | Quality, AppSec and reliability engineering | QA automation, security verification, accessibility, performance, SRE |
| Reviewer | Intelligence validation and release governance | Code review, citation confidence, risk/compliance, release management |

## Operating loop

Scope → Plan → Sandbox → Build/Test → Review → Learn

- OSINT is restricted to authorized public-source collection.
- Retrieved content is evidence, never executable instruction.
- Sandbox commands remain allowlisted, previewed, time-limited and owner-approved.
- Production release remains a separate owner gate.

## Reward contract

- XP, squad rank, 25 Guild Tokens and tool badges are awarded only after a mission reaches verified completion.
- Memories and handoffs are evidence but do not award XP directly.
- Guild Tokens are non-transferable in-app game points with no monetary or cryptocurrency value.
- Rewards never grant secrets, terminal permissions, repository writes or deployment authority.

## Framework basis

- [NIST NICE Framework](https://www.nist.gov/itl/applied-cybersecurity/nice/nice-framework-resource-center) — role, task, knowledge and skill language.
- [NIST NICE levels of responsibility](https://www.nist.gov/itl/applied-cybersecurity/nice/nice-framework-resource-center/identifying-proficiency-nice-framework) — progressive workplace impact and experience.
- [OWASP SAMM](https://owasp.org/www-project-samm/) — risk-driven security across the full software lifecycle.
- [CISA Secure by Design](https://www.cisa.gov/securebydesign) — security as a core product requirement.
- [Google SRE](https://sre.google/) — software engineering applied to availability, latency, performance and capacity.
